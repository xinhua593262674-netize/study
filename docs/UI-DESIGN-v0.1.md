# 教研解析平台 UI 设计文档 v0.1

## 1. 设计目标

本设计服务于一级建造师教研人员，首期聚焦“按真题审核”的高频工作流。

设计目标：

1. 让教研人员在一个屏幕内完成“读题、看 AI 建议、核教材、做判断”。
2. 明确区分 AI 建议、人工审核结果、正式发布数据，避免误判。
3. 对低置信度、规则变化、无直接依据等风险信息保持持续可见。
4. 支持长时间、高密度审核，减少无效跳转和视觉疲劳。
5. 建立可复用的设计语言，覆盖教材验收、批次发布与成果端。

## 2. 信息架构

### 2.1 全局导航

| 一级导航 | 二级入口 | 优先级 |
| --- | --- | --- |
| 审核工作台 | 按真题审核、按知识点审核、按教材章节审核、复审队列 | P0 |
| 数据准备 | 教材解析验收、知识体系差异、真题导入与查重 | P0 |
| 发布管理 | 待发布数据、发布批次、更正记录 | P0 |
| 教研成果 | 教研大宽表、教材预览、知识图谱 | P1/P2 |

### 2.2 核心用户流程

`选择待审真题 -> 逐个最小考查单元审核 -> 核验知识点与教材依据 -> 处理异常 -> 提交初审/复审 -> 进入待发布 -> 批次发布`

## 3. P0 页面设计

### 3.1 按真题审核工作台

采用“固定框架 + 三栏工作区 + 底部操作栏”。

- 左栏 320px：真题队列、筛选、进度和异常标记。
- 中栏自适应：题目正文、选项/得分点、当前审核单元。
- 右栏 480px：AI 关联建议、教材依据、风险项和修改记录。
- 顶部 64px：科目、教材版本、发布版本和全局状态。
- 底部 72px：暂存、退回、提交复审、初审通过。

关键交互：

- 点击题干、选项或得分点，右栏同步切换对应关联。
- AI 建议卡片必须显示置信度、推荐理由和是否已被人工修改。
- 教材依据支持多片段选择，片段按引用顺序编号。
- “无直接教材依据”作为显式状态，不隐藏在备注中。
- 风险项始终显示在提交按钮附近；存在强制复审条件时禁止直接进入待发布。
- 页面离开前自动暂存；手动修改均进入变更记录。

### 3.2 教材解析验收页

采用“原版页面 / 结构化结果”双栏对照。

- 左侧：原版教材页面、页码导航、缩放与区域定位。
- 右侧：标题树、结构化切片、内容类型与解析置信度。
- 底部差异抽屉：知识体系表格与教材标题树差异。

颜色约定：

- 蓝色框：已匹配结构。
- 琥珀色框：低置信度或待确认。
- 红色框：缺失、错序或结构冲突。

### 3.3 待发布与批次发布页

采用“批次概览 + 变更清单 + 检查结果”结构。

- 批次概览：科目、教材版本、真题范围、数据版本。
- 变更清单：新增、修改、停用、待迁移、异常。
- 检查结果：阻断项、警告项、统计变化与三成果端版本一致性。
- 发布为高风险操作，按钮必须显示目标版本和变更总量。

## 4. 成果端设计

### 4.1 教研大宽表

- 冻结知识体系列与表头。
- 标题树支持分组展开，叶子节点独占一行。
- 统计口径不可合并为单一“考查次数”。
- 点击行在右侧抽屉展示教材切片、真题与审核发布记录。

### 4.2 教材预览

- 中心区域保持教材原始阅读体验。
- 精确原文使用下划线标注，不大面积覆盖文字。
- 蓝色代表核心考查，灰蓝代表一般涉及，灰色虚线代表历史失效。
- 点击标注后从右侧展开真题明细。

### 4.3 知识图谱

- 默认只显示与当前节点强相关的局部图谱。
- 关系类型通过线型和标签表达，不只依赖颜色。
- 节点详情同时展示组合考查题数与关联叶子对数。

## 5. 视觉规范

### 5.1 设计关键词

`可信`、`高密度`、`可追溯`、`克制`、`长时间可读`

### 5.2 色彩

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 品牌主色 | `#3159D9` | 主按钮、当前状态、核心关联 |
| 深色导航 | `#17213A` | 全局导航和高层级信息 |
| 页面底色 | `#F4F6FA` | 降低长时间工作的视觉压力 |
| 卡片底色 | `#FFFFFF` | 内容区 |
| 成功 | `#15966A` | 已通过、可发布 |
| 警告 | `#D88724` | 低置信度、待复审 |
| 危险 | `#D84A4A` | 阻断项、冲突 |
| 正文 | `#253047` | 主要文字 |
| 次级文字 | `#6B7487` | 辅助信息 |

### 5.3 字体与密度

- 中文：`PingFang SC`、`Microsoft YaHei`、系统无衬线字体。
- 页面标题 18px/600，区块标题 14px/600，正文 14px/400，辅助文字 12px/400。
- 基础间距单位 4px；常用间距 8、12、16、24px。
- 表单与按钮高度以 32px、36px 为主，避免普通后台常见的过度留白。

### 5.4 组件状态

- 标签：用于题型、审核状态、有效性、关联角色。
- 置信度：数字百分比 + 颜色 + 风险文案，不只显示进度条。
- AI 建议卡：蓝色左边线；人工修改后增加“已调整”标记。
- 教材依据片段：编号、原文、页码、归属节点、引用角色必须同时可见。
- 异常提示：警告使用琥珀色，阻断使用红色，并提供解决入口。

## 6. 响应式与适配

- 推荐工作分辨率：1440px 及以上。
- 1280px 以下隐藏左侧题目摘要，仅保留队列和状态。
- 右侧关联面板可切换为抽屉，但提交操作始终固定可见。
- 首期优先桌面 Web，不设计移动端审核能力。

## 7. 可用性与无障碍

- 关键状态不只依赖颜色，配合图标、标签和文案。
- 所有审核操作支持键盘焦点；推荐快捷键：上一题、下一题、接受建议、提交。
- 离开未提交题目时提示，并保留自动暂存。
- 教材原文、AI 建议和人工结果在视觉与语义上明确分区。

## 8. 开发交接重点

1. 页面状态由“当前题目 + 当前最小考查单元 + 当前关联”共同决定。
2. 右栏必须响应题干、选项、案例得分点切换，不可只按整题刷新。
3. 所有修改保存前后快照，支持查看差异。
4. 风险规则由后端返回，前端负责阻断状态和解释展示。
5. 三个成果端统一展示当前发布版本，并可回溯历史版本。

- `textbook-validation.html`：教材解析验收。
- `knowledge-diff.html`：知识体系差异。
- `release.html`：待发布与批次发布。
- `knowledge-table.html`：教研大宽表。
- `textbook-preview.html`：教材预览。
- `knowledge-graph.html`：知识图谱分析。
