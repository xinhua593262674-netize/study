# UI 与开发交接清单 v0.1

## 1. 第一批页面

### P0：按真题审核工作台

目标：完成一整道真题的知识点及教材依据审核。

核心区域：

1. 真题区：题干、选项或案例得分点、来源与原始文件定位。
2. AI 建议区：1 个主要知识点、多个次要知识点及推荐理由。
3. 教材依据区：基础切片、精确原文片段和原版页面定位。
4. 审核操作区：接受、修改、拒绝、补充、提交、复审。
5. 异常区：无直接依据、低置信度、规则变化、争议信息。

必须支持：

- 客观题按题干和选项逐项关联。
- 案例题按得分点逐项关联。
- 跨自然段选择多个连续或不连续的依据片段。
- 错误选项结构化标记。
- 查看 AI 建议与教研修改差异。

### P0：教材解析验收页

目标：教材达到“可关联”状态。

必须支持：

- 对照原版查看标题树和结构化内容。
- 检查标题层级、自然段顺序、表格、图片、公式和流程图。
- 展示知识体系表格与教材标题树差异清单。
- 修正解析错误并记录变更。

### P0：待发布与批次发布页

目标：将审核通过数据统一发布至成果端。

必须支持：

- 按科目、教材版本、真题范围创建批次。
- 展示新增、修改、停用、待迁移和异常数据。
- 展示发布前检查结果及统计变化。
- 发布批次及创建更正批次。

### P1：教研大宽表

目标：以最新版叶子知识节点查看教材和真题关联全貌。

必须支持：

- 标题树分组、筛选、展开和折叠。
- 每个叶子节点一行。
- 查看教材切片、真题明细及多口径统计。
- 导出主表和明细表。

### P1：教材预览

目标：阅读教材时查看精确考查内容和真题明细。

必须支持：

- 原版阅读与精确原文标注。
- 点击标注查看真题总览及明细。
- 区分核心考查、一般涉及和历史失效内容。

### P2：知识图谱分析

目标：发现跨章节考查、知识组合及命题规律。

必须支持：

- 以叶子节点计算组合关系。
- 上级节点汇总展示。
- 区分共同核心、主次组合、选项对比和案例场景共现。
- 同时展示组合考查题数与关联叶子对数。

## 2. 第一批核心接口能力

- 导入并解析教材原版文件。
- 导入知识体系表格并生成标题树差异。
- 导入真题、查重、结构化题目及最小考查单元。
- 生成 AI 知识点及教材依据建议。
- 保存教研审核结果及变更快照。
- 计算异常条件并进入复审。
- 创建、检查和发布数据批次。
- 查询最新版叶子节点宽表数据。
- 查询教材原文标注及关联真题。
- 计算并查询知识组合关系。

## 3. 建议开发里程碑

### M1：数据底座

- 教材版本、标题树、基础切片与原版定位。
- 真题、最小考查单元及来源定位。
- 知识关联、依据片段、审核记录和发布版本。

### M2：教材解析验收与客观题审核

- 教材结构差异修正。
- 单选、多选题按选项审核。
- AI 建议及精确教材依据审核。

### M3：案例题与发布流程

- 案例得分点拆分与审核。
- 复审、批次发布及更正流程。

### M4：成果端

- 教研大宽表。
- 教材预览。
- 知识图谱分析。

## 4. 首轮验收用例

1. 一个自然段同时被两个知识节点引用，但只有一个主归属节点。
2. 一道多选题包含主要知识点、次要知识点及多个干扰项考查。
3. 一个案例子问题被拆成多个得分点，并关联不同教材依据。
4. 一条教材依据跨多个自然段且包含不连续片段。
5. 一道历史真题在最新版教材中发生规则变化并进入复审。
6. 一个叶子节点新增子节点，原关联进入待拆分队列。
7. 同一道真题的疑似重复版本经教研确认后合并。
8. 已发布关联发现错误，通过更正批次修复且旧版本可追溯。
9. 教材预览标注、宽表统计与知识图谱使用同一发布版本。
10. 上级节点组合关系同时展示去重题数和关联叶子对数。

## 5. UI 与开发启动前仍需输入

- 第一期开工科目名称。
- 最新版教材原版文件。
- 现有知识体系表格。
- 历史真题原始资料。
- 至少一组客观题和一组案例题标准样例。
